
## Гайд: Локальный запуск Flux для Faceless конвейера

Для оптимальной работы на видеокарте **Radeon RX 6900 XT (16GB VRAM)** необходимо использовать легковесную модель Flux, чтобы избежать OOM (Out of Memory) ошибок и добиться высокой скорости генерации.

### Рекомендуемая модель и формат:
*   **Модель:** `Flux.1-schnell` (версия schnell генерирует качественные изображения всего за 4 шага, что идеально для потокового создания Shorts).
*   **Формат:** Обязательно скачивайте квантованную версию **FP8** или **GGUF** (например, `flux1-schnell-fp8.safetensors` или Q8_0/Q5_1 GGUF). Обычная FP16 версия потребует ~24GB VRAM, что превышает лимиты RX 6900 XT. FP8 версия потребляет около 10-12GB VRAM, оставляя запас для системы и браузера.

### Установка в ComfyUI:
1.  Скачайте `flux1-schnell-fp8.safetensors` (весом около 11GB) с HuggingFace (репозитории от Kijai, city96 или аналогичные квантованные версии).
2.  Поместите файл в папку `ComfyUI/models/unet/` или `ComfyUI/models/checkpoints/` (в зависимости от вашей схемы подключения).
3.  Скачайте текстовые энкодеры (Clip L и T5xxl). Для экономии памяти используйте T5xxl в формате FP8. Поместите их в `ComfyUI/models/clip/`.
4.  Скачайте VAE для Flux и поместите в `ComfyUI/models/vae/`.
5.  В ComfyUI используйте ноду `Load Diffusion Model` (выбрав FP8 вес), подключите `DualCLIPLoader` и `VAELoader`.
6.  При генерации выставляйте `steps: 4` и `sampler: euler`.

## Production n8n Workflow (v2)
Сценарий автоматизации был переписан с нуля в файле `/automation/n8n_production_workflow.json` в соответствии с официальными best practices от n8n.

**Внесенные архитектурные фиксы:**
1. **Безопасная авторизация (Credentials):** Вместо хардкодинга ключей `{{$env.API_KEY}}` напрямую в HTTP Request Headers, теперь используются стандартизированные сущности n8n (Credentials Nodes). Это предотвращает утечку токенов в логах выполнения (executions).
2. **Отказоустойчивый парсинг (Code Node):** Стандартный `JSON Parse` заменен на кастомный JavaScript узел (Code Node). Теперь он содержит `try...catch` блок и регулярные выражения, которые автоматически вырезают лишние Markdown-артефакты (например, ` ```json `), которые часто выдают LLM, предотвращая падение парсера.
3. **Обработка ошибок (Error Trigger):** Добавлен независимый флоу с нодой `Error Trigger`. Если API Gemini выдает таймаут или таблицу нельзя прочитать, воркфлоу не зависнет молча, а перехватит ошибку и отправит алерт (по умолчанию настроено на Telegram).
4. **Легитимная запись файлов:** Заглушка заменена на официальный узел `Read/Write Files from Disk` (Local File System), который корректно сохраняет результаты с уникальными Timestamp-именами.

### Инструкция по деплою (в один клик)
1. Откройте ваш веб-интерфейс n8n (обычно `localhost:5678`).
2. В левом меню нажмите на **Workflows** -> **Add Workflow** (кнопка +).
3. В правом верхнем углу нажмите меню с тремя точками (`...`) -> **Import from File**.
4. Выберите файл `/automation/n8n_production_workflow.json` из этого репозитория.
5. После импорта n8n подсветит ноды, требующие авторизации (Google Sheets, Gemini API). Нажмите на них и создайте Credentials, введя ваши ключи.
6. Нажмите **Save** и включите переключатель **Active** в правом верхнем углу.

### Как включить API в ComfyUI
Чтобы n8n мог локально управлять генерацией картинок, вы должны запускать ComfyUI с разрешением на прием внешних запросов:
1. Откройте ваш `run_nvidia_gpu.bat` или стартовый bash скрипт.
2. Добавьте флаг `--enable-cors-header` к аргументам запуска. Пример: `python main.py --enable-cors-header`.
3. Убедитесь, что порт 8188 активен и брандмауэр/антивирус не блокирует локальные HTTP POST запросы на `http://localhost:8188/prompt`.
