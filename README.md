# 🌱 Искра и Росток — Автономный конвейер публикаций

Добро пожаловать в центр управления контентом «Искра и Росток»! Этот проект содержит автономный конвейер для обработки баз знаний, генерации экспертных статей (в стиле Аналитик/Ботаник) и их публикации в Telegram и Яндекс.Дзен.

Ниже представлен мануал по локальному развертыванию и управлению MVP-версией конвейера.

---

## 🛠 Первичная настройка (Env-переменные)

Система отправки постов в Telegram требует ключи доступа.
В корневой папке проекта создайте файл `.env` (без имени, только расширение) и пропишите в него следующие строки:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота_от_BotFather
TELEGRAM_CHAT_ID=-100xxxxxxxxxx_id_вашего_канала
```
*Примечание: скрипты не требуют установки сторонних пакетов (`pip install`), используется чистый системный Python.*

---

## 🚀 Как работает конвейер (Пошаговый Гайд)

### Этап 1: Оркестровка (Сбор знаний)
Скрипт сканирует папку `/knowledge`, находит необработанные `.md` файлы (проверяя `content_state.json`) и ставит их в очередь на генерацию.
**Запуск:**
```bash
python src/botany_orchestrator.py
```
*(Результат: в папке `/drafts` появляются новые черновики статей).*

### Этап 2: Визуальный Апрув и Дзен (Экспорт)
Автоматическая генерация картинок нейросетями отключена. Даша генерирует их локально на RX 6900 XT, используя промпты из блока `## Галерея для публикации` в конце каждого черновика.
Чтобы удобно прочитать статью и скопировать ее в Яндекс.Дзен, используйте локальный экспортер:
**Запуск:**
```bash
python src/exporter.py article_05_pest_control.md
```
*(Результат: в папке `/drafts/preview` появится красивый `.html` файл. Откройте его в браузере).*

### Этап 3: Публикация в Telegram (Шедулер)
Как только статья проверена, а картинки (опционально) лежат рядом, запускайте шедулер. Он найдет все статьи со статусом `"drafted"` в `content_state.json`, отрежет технический блок "Галерея" и отправит пост напрямую в ваш Telegram-канал.
**Запуск:**
```bash
python src/scheduler.py
```
*(Результат: Пост выходит в Telegram, а статус статьи в `content_state.json` меняется на `"published"`, платформа `"tg": true`).*
